# 智能迁移系统使用指南

## 🎯 系统概述

智能迁移系统是一个**策略驱动的自动建造系统**，用于处理现有房间与新布局之间的建筑迁移。系统采用**配置策略 + 自动执行**的设计，提供安全、智能、可控的房间重构方案。

## 🔄 系统架构

### **核心设计理念**
- **默认自动**: 系统每50tick自动执行建造，无需手动干预
- **策略配置**: 通过命令灵活配置建造策略
- **统一建造**: 所有模式使用相同的优先级建造顺序
- **可选拆除**: 根据策略决定是否拆除多余建筑

## 📋 核心功能

### 1. **智能分析**
- 自动分析现有建筑与布局的差异
- 评估迁移风险等级
- 提供智能建议

### 2. **策略驱动建造**
- **AUTO模式**: 默认模式，按优先级建造缺失建筑，不拆除任何建筑
- **SAFE模式**: 保守策略，只建造缺失建筑，保留所有现有建筑
- **SMART模式**: 智能优化，建造缺失建筑 + 智能拆除多余低价值建筑
- **AGGRESSIVE模式**: 激进重建，建造缺失建筑 + 拆除所有多余建筑 + 处理错位建筑

### 3. **安全保障**
- 自动备份房间状态
- 安全检查机制
- 一键回滚功能

## 🚀 快速开始

### 基本流程


## 📊 详细命令说明

### 分析命令

#### `analyzeMigration(roomName)`
分析房间迁移需求，显示详细的差异报告。

**输出示例:**
```
📊 房间 E59N32 迁移分析:
├─ 缺失建筑: 5
├─ 错位建筑: 3
├─ 多余建筑: 8
├─ 正确建筑: 12
├─ 风险等级: MEDIUM
├─ 💡 建议:
│  └─ 建议分批进行迁移
└─ 🔍 缺失建筑详情:
   └─ extension: 3
   └─ tower: 2
```

### 策略配置命令

#### `migrateRoom(roomName, strategy)`
**配置房间建造策略**，系统将自动执行。支持四种策略：

**策略说明:**
- **'AUTO'**: 默认模式：按优先级建造缺失建筑，不拆除任何建筑
- **'SAFE'**: 保守策略：只建造缺失建筑，保留所有现有建筑
- **'SMART'**: 智能优化：建造缺失建筑 + 智能拆除多余低价值建筑（推荐）
- **'AGGRESSIVE'**: 激进重建：建造缺失建筑 + 拆除所有多余建筑 + 处理错位建筑

**使用示例:**
```javascript
// 自动模式 - 默认行为
migrateRoom('E59N32', 'AUTO')

// 安全模式 - 适合新手或关键房间
migrateRoom('E59N32', 'SAFE')

// 智能模式 - 推荐大多数情况
migrateRoom('E59N32', 'SMART')

// 激进模式 - 完全重建
migrateRoom('E59N32', 'AGGRESSIVE')
```

**输出示例:**
```javascript
🎯 配置房间 E59N32 建筑策略: SMART
✅ 策略配置成功
💾 自动备份ID: migration_backup_E59N32_12345678
⏱️ 自动建造系统将在下次tick执行新策略
📋 智能模式 (🟡 低风险):
   智能优化：建造缺失建筑 + 智能拆除多余低价值建筑
```

### 策略管理命令

#### `getStrategy(roomName)`
查看房间当前策略配置

**输出示例:**
```javascript
📋 房间 E59N32 当前策略:
├─ 模式: SMART
├─ 状态: 启用
├─ 上次迁移: 2024-03-27 18:00:00
└─ 备份ID: migration_backup_E59N32_12345678
```

#### `resetStrategy(roomName)`
重置房间为AUTO模式

#### `pauseBuilding(roomName)`
暂停房间自动建造

#### `resumeBuilding(roomName)`
恢复房间自动建造

### 备份管理

#### `listBackups()`
列出所有迁移备份

#### `restoreBackup(backupId)`
恢复指定备份

#### `cleanupBackups()`
清理过期备份

## 🛡️ 安全机制

### 自动备份
每次迁移前自动创建备份，包含：
- 所有现有建筑信息
- 所有工地信息
- 时间戳记录

### 安全检查
迁移前进行多项安全检查：
- **Spawn检查**: 确保不会拆除唯一的spawn
- **资源检查**: 检查storage/terminal中的资源
- **能量检查**: 确保房间能量充足

### 智能判断
系统会智能判断哪些建筑应该拆除：
- **低价值建筑**: 道路、container、rampart
- **位置不佳**: 阻碍路径、孤立、偏远
- **效率低下**: 长期空着、不活跃
- **重复建筑**: 超过合理数量的建筑

## 📈 风险评估

### 风险等级

| 等级 | 描述 | 建议 |
|------|------|------|
| **LOW** | 差异很小 | 可以激进迁移 |
| **MEDIUM** | 有一些差异 | 建议智能迁移 |
| **HIGH** | 差异较大 | 建议分批迁移 |
| **CRITICAL** | 有关键建筑冲突 | 必须谨慎处理 |

### 关键建筑
以下建筑被视为关键建筑：
- **Spawn**: 房间生产核心
- **Storage**: 资源存储中心
- **Terminal**: 资源交易中心

## 🔧 高级用法

### 策略切换管理
根据房间状态动态调整策略：

```javascript
// 1. 先分析现状
analyzeMigration('E59N32')

// 2. 根据风险等级选择策略
if (风险等级 === 'CRITICAL') {
    // 关键冲突，先安全建造
    migrateRoom('E59N32', 'SAFE')
} else if (风险等级 === 'HIGH') {
    // 高风险，智能优化
    migrateRoom('E59N32', 'SMART')
} else {
    // 低风险，可以激进重建
    migrateRoom('E59N32', 'AGGRESSIVE')
}

// 3. 监控执行状态
getStrategy('E59N32')
```

### 暂停和恢复
在特殊情况下暂停自动建造：

```javascript
// 战争期间暂停建造
pauseBuilding('E59N32')

// 战争结束后恢复
resumeBuilding('E59N32')
```

### 分阶段优化
对于复杂房间的渐进式优化：

```javascript
// 阶段1: 只建造不拆除
migrateRoom('E59N32', 'SAFE')

// 等待建造完成（观察日志）
// 阶段2: 智能优化
migrateRoom('E59N32', 'SMART')

// 如需要，阶段3: 激进重建
migrateRoom('E59N32', 'AGGRESSIVE')
```

### 批量房间管理
对多个房间统一配置策略：

```javascript
// 批量设置智能模式
['E59N32', 'E58N33', 'E60N31'].forEach(room => {
    migrateRoom(room, 'SMART')
});

// 批量查看策略状态
['E59N32', 'E58N33', 'E60N31'].forEach(room => {
    getStrategy(room)
});
```

## ⚠️ 注意事项

### 迁移前检查
1. **确保房间安全**: 不在危险状态下迁移
2. **检查能量储备**: 建议能量 > 70%
3. **确认布局正确**: 使用VP命令可视化布局
4. **备份重要数据**: 虽然系统自动备份，但建议额外确认

### 迁移后验证
1. **检查关键建筑**: 确保spawn、storage正常
2. **验证能量流**: 确保能量供应正常
3. **观察creep行为**: 确保creep能正常工作
4. **监控CPU使用**: 迁移可能增加CPU消耗

### 常见问题

**Q: 迁移过程中断怎么办？**
A: 系统有自动备份，可以使用`restoreBackup()`回滚。

**Q: 可以部分迁移吗？**
A: 可以使用SAFE模式只建造不拆除，或者手动处理特定建筑。

**Q: 迁移会影响creep吗？**
A: 短期内可能影响，但系统会尽量保持关键建筑可用。

**Q: 如何撤销迁移？**
A: 使用`listBackups()`查看备份，然后用`restoreBackup()`恢复。
