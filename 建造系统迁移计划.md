### 第二阶段：桥接增强 (2-3天)

#### 2.1 增强 `buildFromLayout()` 函数
**当前位置**: `rooms.construction2.ts` 第619行

**所需更改**:
```typescript
// 增强版本以支持传统角色
function buildFromLayout(room: Room, targetStructureType?: string, maxTargets?: number): ConstructionSite[] | null {
    if (!Memory.roomPlanner || !Memory.roomPlanner[room.name]) {
        return null; // 回退到传统系统
    }

    const layout = Memory.roomPlanner[room.name].layout;
    const existingSites = room.find(FIND_MY_CONSTRUCTION_SITES);
    const existingStructures = room.find(FIND_MY_STRUCTURES);

    // 如果指定了结构类型则过滤（传统角色兼容性）
    let targets = [];
    for (const [structureType, positions] of Object.entries(layout)) {
        if (targetStructureType && structureType !== targetStructureType) continue;

        for (const pos of positions) {
            const roomPos = new RoomPosition(pos.x, pos.y, room.name);

            // 检查是否已建造或正在建造
            const hasStructure = existingStructures.some(s =>
                s.pos.x === pos.x && s.pos.y === pos.y && s.structureType === structureType
            );
            const hasSite = existingSites.some(s =>
                s.pos.x === pos.x && s.pos.y === pos.y && s.structureType === structureType
            );

            if (!hasStructure && !hasSite) {
                targets.push({ structureType, pos: roomPos });
            }
        }
    }

    // 如果指定了则限制结果数量
    if (maxTargets && targets.length > maxTargets) {
        targets = targets.slice(0, maxTargets);
    }

    return targets;
}
```

#### 2.2 创建布局感知目标查找器
**新函数**: `findLockedFromLayout()`

```typescript
function findLockedFromLayout(creep: Creep): string | null {
    const room = creep.room;
    const layoutTargets = buildFromLayout(room);

    if (!layoutTargets || layoutTargets.length === 0) {
        return null; // 回退到传统系统
    }

    // 应用类似当前 findLocked() 的优先级逻辑
    const prioritizedTargets = prioritizeTargets(layoutTargets, creep);

    if (prioritizedTargets.length > 0) {
        creep.memory.suicide = false;
        creep.say("布局", true);
        return prioritizedTargets[0].pos.createConstructionSite(prioritizedTargets[0].structureType) === OK
            ? prioritizedTargets[0].id
            : null;
    }

    creep.memory.suicide = true;
    return null;
}
```

### 第三阶段：核心角色迁移 (3-4天)

#### 3.1 迁移 `builder.ts` (优先级：高)

**当前问题**:
- 使用硬编码优先级系统
- 对房间布局无感知
- 简单的 `progressTotal` 排序

**迁移步骤**:
1. **备份当前 `findLocked()` 函数** 为 `findLockedLegacy()`
2. **用混合方法替换 `findLocked()`**:
```typescript
function findLocked(creep) {
    // 首先尝试新布局系统
    if (creep.room.memory.layoutEnabled !== false) {
        const layoutTarget = findLockedFromLayout(creep);
        if (layoutTarget) return layoutTarget;
    }

    // 回退到传统系统
    return findLockedLegacy(creep);
}
```
3. **添加布局感知优先级逻辑**
4. **使用不同房间配置进行测试**

#### 3.2 迁移支持角色

**`energyMiner.ts`**:
- 更新工地检测以尊重布局
- 保持现有的link/container放置逻辑

**`remoteBuilder.ts`**:
- 与布局系统集成进行容器放置
- 保留spawn建造逻辑

**`carry.ts` & `Dismantler.ts`**:
- 更新 `findLocked()` 实现

## 风险评估与缓解

### 高风险
1. **建造停滞**: 新系统可能不创建必要工地
   - **缓解**: 回退到传统系统
   - **监控**: 跟踪每个房间的工地数量

2. **角色故障**: 角色可能找不到目标
   - **缓解**: 过渡期间使用混合方法
   - **监控**: 跟踪角色有效性指标

3. **性能问题**: 新系统可能更慢
   - **缓解**: 缓存和优化
   - **监控**: CPU使用跟踪

### 中等风险
1. **布局冲突**: 新布局可能与现有结构冲突
   - **缓解**: 迁移分析工具
   - **监控**: 结构不匹配跟踪

2. **房间状态不一致**: 部分迁移状态
   - **缓解**: 原子迁移操作
   - **监控**: 房间状态验证
